# خطة تطوير وحدة التخطيط والتنبؤ الذكي (AI Planning Hub)

## الهدف العام
إعادة تصميم وتطوير `PlanningHub.cshtml` من واجهة تبويبات بسيطة إلى **مساحة عمل متكاملة (Full Page Workspace)** عالية التفاعل تتبع نظام "مناطق التدفق المتسلسل" وتُعبّر عن سير العمل الكامل من التنبؤ حتى الاستلام.

---

> [!IMPORTANT]
> هذا التطوير يقتصر على **الواجهة الأمامية (Frontend)** بالكامل مع بيانات وهمية (Mock Data) للعرض. لا يوجد تغيير على قاعدة البيانات أو الكنترولر في هذه المرحلة.

---

## الهيكل التقني المعتمد
- **التصميم:** Tailwind CSS + Vanilla JS (لا تغيير على التقنية)  
- **الرسوم البيانية:** Chart.js (موجودة مسبقاً)
- **القالب:** `_Layout.cshtml` (الموجود)
- **ملف الهدف:** `PlanningHub.cshtml` (إعادة كتابة شاملة)
- **قائمة التنقل:** تحديث `NavigationService.cs` لجعلها توجيهاً مباشراً لصفحة واحدة

### 🔵 المنطقة A — شريط الحالة العلوي (Status Bar)
> **شريط Stepper** في أعلى الصفحة يوضح مرحلة الخطة الحالية

```
[1: تنبؤ آلي] → [2: مراجعة المدير] → [3: بانتظار تأكيد PO] → [4: استلام البضاعة] → [5: ترحيل مالي ✓]
```
- مؤشر بصري يُبرز المرحلة النشطة (رقم ملوّن + نبضة خضراء)
- كل خطوة قابلة للنقر لعرض تفاصيلها

---

### 🟡 المنطقة B — لوحة تحكم محرك الذكاء الاصطناعي (AI Engine Control Panel)
> بطاقة شبه-شفافة (glassmorphism) تحمل 3 أقسام جانبية:

#### 1. مصدر البيانات:
- زر **"تحديث من قاعدة البيانات"** (يشغّل animation وهمياً)
- منطقة **Drag & Drop** لرفع ملف CSV/Excel مع عرض اسم الملف
  
#### 2. اختيار المحرك (Model Selection):
بطاقتان قابلتان للاختيار:
```
┌─────────────────────┐    ┌─────────────────────┐
│ 🤖 Prophet         │    │ ☁️ Vertex AI        │
│ معالجة محلية       │    │ تحليل سحابي متقدم   │
│ سريع • خاص        │    │ دقيق • يتطلب اتصال  │
└─────────────────────┘    └─────────────────────┘
```

#### 3. أفق التنبؤ الزمني:
شريط تمرير تفاعلي (Slider) بـ 4 نقاط:
```
[شهر] ─────●───── [ربع سنة] ─────── [نصف سنة] ─────── [سنة]
```
عند تحريك الشريط، تتغير بيانات الرسم البياني في المنطقة التالية.

### 🟢 المنطقة C — التحليل البصري وشفافية الخوارزمية (Visual Analytics)
**قسمان جانبيان (Side by Side):**

#### القسم الأيمن (70%): مخطط الطلب التفاعلي
- Chart.js Line Chart مع بيانات تاريخية (خط أزرق صلب) + تنبؤ (خط برتقالي منقط)
- يتمدد أو يتقلص بناءً على Slider الأفق الزمني
- Tooltip تفاعلي يُظهر القيم عند التمرير فوق النقاط

#### القسم الأيسر (30%): لوحة شفافية الخوارزمية
> زر **"كواليس الذكاء الاصطناعي"** (متلألئ ✨) يفتح Panel منزلق يشرح:

```
المراحل السبع للمحرك:
━━━━━━━━━━━━━━━━━━━━━━━━
✅ 1. تجميع المعطيات
✅ 2. تصنيف ABC
```

### 🔴 المنطقة D — مسودة المشتريات الذكية (Smart Draft Grid)
> جدول البيانات التفاعلي الرئيسي

| ☑ | الصنف | ABC | الرصيد | التنبؤ | EOQ المقترح | **الكمية المعتمدة** | الحالة المالية |
|---|-------|-----|--------|--------|-------------|---------------------|----------------|
| ✓ | بنادول | A | 120 | 340 | 200 | **[حقل إدخال]** | ✅ ضمن السيولة |
| ✓ | أموكسيسيلين | B | 45 | 290 | 245 | **[حقل إدخال]** | ⚠️ عجز جزئي |
| ✓ | فنتولين | A+ | 8 | 150 | 142 | **[حقل إدخال]** | 🔴 عجز كامل |

**ميزات عمود "الحالة المالية":**
- مؤشر شريطي ملوّن (Progress Bar) يُظهر نسبة المتاح من الميزانية
- ألوان: أخضر (ضمن السيولة) / برتقالي (عجز جزئي) / أحمر (عجز كامل)

**أسفل الجدول — شريط الأدوات النهائي:**
```
[🔀 قاطع الأمان: الأدوية المنقذة للحياة] ←ON/OFF Toggle→  ║  [📋 تصدير PDF مخصص] ║ [✅ اعتماد وتحويل لـ PO]
```

- **قاطع الأمان (Safety Switch):** عند التفعيل:
  - تُضاف class خاصة لصفوف الأدوية الحيوية (border أحمر + شارة "♥ حيوي")
  - تُضاعف كمياتها تلقائياً في حقل الإدخال

- **زر "اعتماد وتحويل لـ PO":** يُظهر Modal تأكيد جميل (SweetAlert2) ثم يتحول الـ Stepper للمرحلة الثالثة

### ⚫ المنطقة E — بطاقات طلبات التوريد المعتمدة (PO Cards)
> تظهر فقط بعد الاعتماد (تنزلق من الأسفل بـ animation)

لكل مورد بطاقة تحتوي:
- شريط Stepper الخاص به (انتظار → مستلم جزئياً → مستلم كلياً)
- زر **"تصدير PDF ذكي"** → يفتح Modal لاختيار الأعمدة + خيار مشاركة WhatsApp
- زر **"استلام البضاعة"** → يفتح نافذة لملء: السعر الفعلي + رقم التشغيلة + تاريخ الصلاحية لكل صنف

## المرحلة 4 — منطق استلام البضاعة (Receipt Modal)

> Modal ينبثق عند الضغط على "استلام"، يحتوي على:
> - قائمة أصناف الطلب (مع بيانات الكمية المطلوبة مُعبأة)
> - 3 حقول إجبارية لكل صنف: السعر + Batch + Expiry Date
> - مؤشر FEFO: ⚠️ إذا كان تاريخ الصلاحية قريباً أو يتعارض مع مخزون أقدم
> - عند الحفظ: Toast "تم الاستلام والترحيل ✓"
